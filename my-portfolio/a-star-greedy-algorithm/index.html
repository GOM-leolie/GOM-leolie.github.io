Uni Project: A* vs Greedy Shortest Path Algorithm

Comparing two populars shortest path algorithm to find out the pro and cons of each and how they perform against each other.
Project period: December 2014 - March 2015.

Background

This project was created in the collaboration with the a group of student from Indonesia as part of university's final project in artifical intelligence field. The project was initiated by the Indonesian students, consisting of 3 people, and myself. They were responsible for the project documentation, including specifying requirements, testing, and writing the report. I was responsible to write the code and developing the software and implementing the algorithm.

The goal was to measure the effectiveness and efficiency of two most popular shortest path algorithm, A* and Greedy, by measuring the time needed to find the shortest path and result of their findings. The program was developed with C# as the back-end language and WPF for the interface framework.

A* algorithm is a shortest-path algorithm that considered one of the most effective and efficient, pending the accuracy of the heuristic cost. It works by calculating the heuristic cost from the source to the destination, along with the actual cost. Heuristic cost is a straight line distance from source to the destination. Greedy algorithm is known as a fast-processing algorithm. Although it does not always providing the best solution, the algorithm itself has a O(N) time complexity. It works by choosing the next destination with minimum cost until it reached the final destination.

Process and Implementation

This project utilised Central Java, a province in Indonesia, as the graph. Each node represents a major city in that province and the distance between each node was the actual distance at the time in KM. This project assumes the supplied heuristic value is true and accurate at the time. The user needs to specify the start city and destination city. The program will determine the shortest path between these cities by using A* algorithm and Greedy algorithm. Once finished, the program will display the total distance (in KM) of the shortest path calculated by both algorithms, along with the time required to finish the process (in ms).

Simulation

The simulation starts from the City of Cilacap (Node C) with the destination city Purwodadi (Node L).

As seen, greedy algorithm finished significantly faster than it's counter part by 33%, however in term of costing it does not return the optimum solution.

Although A* algorithm performs slower, it was able to found the shortest path from start to destination.

Findings

Both of the algorithms have their own strength and weakness and will outperform each other depending on the situation.

For the larger informed graph with the complex node, greedy algorithm will be a better solution because even though it does not provide optimum solution, it still provides a relatively good solution with much faster processing time.

However if finding an optimum solution is a must, then A* algorithm can help you achieve that.

Conclusion

After running test and several simulation, we drew a conclusion that although Greedy algorithm does not always give us an optimum solution, it always give us a better speed then A*. This algorithm works better in a large graph with complex node.

On the other hand, the A* algorithm always produce an optimum solution, means it is always able to find the shortest path. This algorithm will also works better in the smaller graph, otherwise it will affect the speed of the operation.

Both of the solutions will require a heuristic cost as accurate as possible.
